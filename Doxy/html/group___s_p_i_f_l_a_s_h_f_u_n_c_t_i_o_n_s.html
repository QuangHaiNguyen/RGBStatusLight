int8_t spi_flash_erase ( uint32_t  u32Offset,
uint32_t  u32Sz 
)

Erase a specified range of sectors within the SPI flash.

Parameters
[in]u32OffsetAddress offset within the SPI flash to erase from.
[in]u32SzTotal size of data to be erased (in bytes).
Note
No firmware is required to be loaded on the WINC for the SPI flash to be accessed.
The SPI flash is erased in whole sectors, therefore if u32Offset is not aligned with a sector boundary it will rounded down to the nearest sector start address. If the erase range ends before a sector boundary it will be rounded up to include a complete sector.
Warning
The read must not exceed the last address of the flash.
If the WINC device has running firmware it must be stopped before interacting with the SPI flash using m2m_wifi_download_mode.
See also
m2m_wifi_download_mode
Returns
The function returns M2M_SUCCESS for successful operations and a negative value otherwise.

Write a specified block of data to the SPI flash.

Parameters
[in]pu8BufPointer to a data buffer containing data to be written.
[in]u32OffsetAddress offset within the SPI flash to write the data to.
[in]u32SzTotal size of data to be written (in bytes).
Note
No firmware is required to be loaded on the WINC for the SPI flash to be accessed.
Warning
The read must not exceed the last address of the flash.
If the WINC device has running firmware it must be stopped before interacting with the SPI flash using m2m_wifi_download_mode.
The flash sector should be erased before writing data using the function spi_flash_erase otherwise the correct data will not be written.

m2m_wifi_download_mode

Returns
The function returns M2M_SUCCESS for successful operations and a negative value otherwise.
